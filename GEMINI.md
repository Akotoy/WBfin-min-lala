# AGENT INSTRUCTIONS — WBfin

> Этот файл автоматически читается AI-агентом при старте сессии.
> Это ОБЯЗАТЕЛЬНЫЕ правила. Нарушение = провал задачи.

## Проект
SPA-дашборд финансовой аналитики для продавцов Wildberries.
**Стек:** React 18 + Vite + TypeScript + TailwindCSS + shadcn/ui + Supabase + Vercel.

---

## ⛔ ЖЁСТКИЕ ЗАПРЕТЫ (нарушение = откат изменений)
1. **НЕ обращаться к WB API напрямую из браузера.** Только через `api/wb-proxy.ts`.
2. **НЕ использовать localStorage.** Только Supabase.
3. **НЕ добавлять `"framework": "vite"` в vercel.json.** Убьёт Serverless Functions.
4. **НЕ коммитить `.env`.** Ключи — только в Vercel Environment Variables.
5. **Алиас `@/` = корень проекта**, НЕ `src/`.

## 🔄 АВТОМАТИЧЕСКИЙ ПРОТОКОЛ РАБОТЫ

### ПРИ СТАРТЕ каждой сессии (ОБЯЗАТЕЛЬНО, без исключений):
```
1. Прочитать memory/index.md     → понять контекст проекта
2. Прочитать memory/bugs.md      → не повторять решённые баги
3. Прочитать memory/changelog.md → понять что уже сделано и что в TODO
```

### ПРИ ЗАВЕРШЕНИИ каждой задачи (ОБЯЗАТЕЛЬНО, без исключений):
Перед тем как ответить пользователю «готово», агент ОБЯЗАН выполнить этот чеклист:

```
□ npm run build                            → Сборка не падает?
□ memory/changelog.md обновлён             → Что сделал, что осталось
□ memory/bugs.md обновлён (если баг)       → Симптом + причина + фикс
□ memory/decisions.md обновлён (если ADR)  → Контекст + решение + последствия
□ memory/architecture.md обновлён          → Если изменилась структура/потоки
□ memory/database.md обновлён              → Если изменилась схема БД
```

Агент НЕ МОЖЕТ сказать «готово» пока все применимые пункты не выполнены.
Если пункт не применим к текущей задаче — пропустить.
Если применим и не выполнен — ВЫПОЛНИТЬ СЕЙЧАС.

### ФОРМАТ ЗАПИСИ В CHANGELOG:
```markdown
## Сессия N — YYYY-MM-DD
### Что сделали:
1. [действие]
### Что НЕ доделали:
- [ ] [задача]
```

### ФОРМАТ ЗАПИСИ БАГА:
```markdown
### BUG-NNN: [Название]
- **Симптом:** [что видит пользователь]
- **Причина:** [что на самом деле сломалось]
- **Фикс:** [что сделали] → ссылка на ADR если есть
```

### ФОРМАТ ЗАПИСИ ADR:
```markdown
## ADR-NNN: [Название]
- **Статус:** Принято / Отклонено / Обсуждается
- **Дата:** YYYY-MM-DD
- **Контекст:** [зачем]
- **Решение:** [что]
- **Последствия:** [плюсы и минусы]
```
