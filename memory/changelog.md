## Сессия 1 — 2026-04-10

### Что сделали:
1. **Переехали на Vercel.** Проект изначально жил в подпапке `WBfin-min-lala`, из-за чего Vercel видел пустой репо. Перенесли всё в корень.

3. **Интегрировали Supabase.** 
   - Установили `@supabase/supabase-js`
   - Создали `lib/supabase.ts`, `components/Auth.tsx`
   - Написали SQL-скрипт для таблиц `user_settings` и `product_cogs`
4. **Переписали App.tsx.**
   - Убрали localStorage, подключили Supabase для хранения настроек и COGS
   - Добавили систему сессий (login/logout)
   - Удалили демо-режим (фейковые данные)
   - Добавили кнопку «Выйти» в сайдбар
5. **Починили баги.**
   - 404 на API-прокси → убрали `framework: "vite"` из vercel.json
   - Регистрация молча падала → добавили feedback-сообщение + отключили Confirm Email
   - Build error на импорте → перенесли файлы в правильные директории

## Сессия 2 — 2026-04-10 (вечер)

### Что сделали:
1. **Полный аудит WB API интеграции.** Нашли 6 критических проблем.
2. **Полная перезапись `wbService.ts`:**
   - Исправлены URL (v1 → v5 для финотчёта)
   - Добавлена **полная пагинация** через `rrdid` для `reportDetailByPeriod` — теперь получаем ВСЕ данные
   - Добавлена **пагинация карточек** через cursor — теперь получаем больше 100 товаров
   - Добавлена **retry-логика** с exponential backoff для 429 (rate limit)
   - Исправлен `updatePrice()` — убран мусорный вызов, правильный URL `discounts-prices-api`
   - Все домены вынесены в константы
3. **Укрепление `wb-proxy.ts`:**
   - Добавлена валидация домена (whitelist — только WB домены)
   - Пробрасываются rate-limit заголовки от WB
   - Корректная обработка 204 (конец пагинации)
   - CORS headers
   - Улучшенные сообщения об ошибках
4. **Создан `memory/wb-api.md`** — полный справочник WB API (домены, токены, endpoints, лимиты, грабли)
5. **Перестроена система memory** по best practices (GEMINI.md с автопротоколом)
