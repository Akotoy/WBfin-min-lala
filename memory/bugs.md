## 🔴 Критические (решённые)

### BUG-001: 404 на /api/wb-proxy (Vercel)
- **Симптом:** Консоль браузера заполнена ошибками `Failed to load resource: 404 /api/wb-proxy`
- **Причина:** В `vercel.json` стоял `"framework": "vite"`, из-за чего Vercel игнорировал папку `api/`
- **Фикс:** Убрать `framework`, явно указать `buildCommand` / `outputDirectory` → [ADR-003](decisions.md#adr-003)

### BUG-002: Регистрация «не работает» (кнопка не реагирует)
- **Симптом:** Нажатие «Зарегистрироваться» ничего не делает визуально
- **Причина:** Supabase требовал Confirm Email. Аккаунт создавался, но сессия не возвращалась. Не было feedback-сообщения для пользователя.
- **Фикс:** 
  1. Добавлено зелёное уведомление о создании аккаунта
  2. Отключен Confirm Email в Supabase
  3. Тестовый пользователь создан через Admin API → [ADR-004](decisions.md#adr-004)

### BUG-003: Импорт `@/lib/supabase` не найден (build error)
- **Симптом:** `Could not load C:\...\WBfin/lib/supabase`
- **Причина:** Файл `supabase.ts` лежал в `src/lib/`, а алиас `@/` резолвится в корень проекта (не в `src/`).
- **Фикс:** Переместить `supabase.ts` в `lib/` (корень проекта) и `Auth.tsx` в `components/` (корень проекта).

## 🔴 Критические (решённые — Сессия 2)

### BUG-004: 401 Unauthorized на content-api.wildberries.ru
- **Симптом:** Логи Vercel показывают `401 "unauthorized"` при загрузке дашборда
- **Причина:** Токен пользователя (тип «Базовый») имеет права Статистика/Финансы, но НЕ имеет прав на «Контент». Функция `getProducts()` вызывала Content API.
- **Фикс:** Удалена зависимость от Content API. Товары теперь собираются из данных статистики (`buildProductsFromData()`) → [ADR-006](decisions.md#adr-006)

### BUG-005: 500 TypeError: fetch failed на wb-proxy
- **Симптом:** Лог `[ual6qo] Stack: TypeError: fetch failed at node:internal/deps...`
- **Причина:** Таймаут или сетевая ошибка при запросе к WB API из Vercel Serverless Function
- **Фикс:** Добавлена retry-логика с exponential backoff в `wbService.ts`. Прокси теперь корректно логирует ошибки с requestId.

### BUG-006: Ошибка авторизации демо-пользователя (Неверный логин/пароль)
- **Симптом:** Пользователь при входе видит ошибку логина или пароля.
- **Причина:** Вероятно пароль `demo@wbfin.ru` был изменен на проде или аккаунт удален.
- **Фикс:** Создан новый тестовый аккаунт `test@wbfin.ru`, доступы обновлены в памяти проекта.
